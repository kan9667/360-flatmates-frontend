STATE MANAGEMENT

Riverpod controllers, not widgets

// DiscoverFeedController
@riverpod
class DiscoverFeedController
extends Notifier<PagedState<PropertyListing>> {
@override
PagedState<PropertyListing> build() {
return PagedState.empty();
}
Future<void> refresh() async {
state = state.copyWith(isLoading: true);
final data = await ref
.read(discoverRepoProvider)
.fetchPage(page: 1);
state = PagedState(items: data);
}
}

Controllers in application/

Named methods like refresh(), loadMore(), toggleFilter(). Widgets call ref.read(controller.notifier).method().

StateProvider for UI state

No setState() in ConsumerStatefulWidget. Local UI state uses file-level StateProvider.

PagedState<T>

Paginated data with initial/refresh/load-more loading states built in.
